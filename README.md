

# **Project Manager App - Summary**

## **Overview**

The Project Manager app is a **client-side web application** that allows users to **create, view, edit, and delete posts**. Each post includes a **title, author, content, creation timestamp, and last edited timestamp**. Posts are **stored locally** so that data persists across browser sessions.

---

## **Key Features**

1. **Create Post** – Add a new post with a title, author, and content.
2. **View Post Details** – See full post information on a dedicated page.
3. **Edit Post** – Modify existing post details; updates the “last edited” timestamp.
4. **Delete Post** – Remove posts with confirmation.
5. **Posts as Cards** – All posts displayed in a summary view with key details.
6. **Persistent Storage** – Posts are saved locally and remain available after closing the browser.
7. **Responsive & User-Friendly** – Intuitive interface with feedback for loading and errors.

---

## **Data Structure**

Each post stores:

* **Title** – The post’s heading.
* **Author** – Name of the post creator.
* **Content** – The main body of the post.
* **Creation Timestamp** – Time the post was created.
* **Last Edited Timestamp** – Updated time if the post is edited.

---

## **Usage Flow**

1. Create a post by entering required details.
2. View posts in a list; select a post to see full details.
3. Edit or delete posts as needed.
4. Changes are automatically saved locally and reflected in the list.

---

## **Deployment**

The app can be **hosted online** using static hosting services, making it accessible via a URL without requiring a backend.

---

## **Future Improvements**

* Implement **search and filter functionality**.
* Enable **multi-user support** and online persistence.
